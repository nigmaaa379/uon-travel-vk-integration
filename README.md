# U-ON Travel × ВКонтакте

Готовый сервис интеграции чат-бота сообщества ВКонтакте с CRM U-ON Travel. После завершения квалификации туриста сервис создаёт карточку именно в разделе **«Обращения»** через метод `lead/create`, затем отправляет мгновенные уведомления на email и в Telegram.

## Реализовано

- [x] Callback API ВКонтакте с проверкой `group_id` и секретного ключа.
- [x] Сценарий квалификации: направление, даты, состав туристов, бюджет, имя, телефон и email.
- [x] Валидация телефона и email.
- [x] Создание **обращения**, а не заявки, через `POST /lead/create.json` U-ON Travel.
- [x] Передача источника «ВКонтакте — чат-бот» и результатов опроса.
- [x] Защита от повторной обработки событий VK.
- [x] Повторные попытки при временной ошибке U-ON Travel.
- [x] Email-уведомление куратору/руководителю через Resend.
- [x] Уведомление в Telegram.
- [x] Хранение состояния диалогов на диске.
- [x] Docker-конфигурация, healthcheck endpoint и автоматические тесты.

## Сценарий чат-бота

1. Пользователь пишет сообществу.
2. Бот последовательно собирает параметры поездки и контакты.
3. После валидного email сервис вызывает U-ON Travel `lead/create.json`.
4. Полученный ID обращения сохраняется, а туристу приходит подтверждение.
5. Куратору/руководителю отправляются email и Telegram-уведомление.
6. Повторное событие VK не создаёт вторую карточку.

Команды `начать`, `старт`, `start`, `поехали` и `заново` запускают опрос заново. До завершения опроса интеграция не обращается к CRM.

## Быстрый запуск

Требуется Node.js 20+ или Docker.

```bash
cp .env.example .env
# заполните .env
npm test
npm start
```

С Docker:

```bash
docker compose up -d --build
curl http://localhost:3000/health
```

## Настройка U-ON Travel

1. Откройте **Настройки → Интеграции → API / Webhooks**.
2. Разрешите POST-запросы API.
3. Скопируйте API-ключ в `UON_API_KEY`.
4. Укажите источник в `UON_SOURCE`.
5. Если поле комментария в вашем аккаунте называется иначе, задайте его API-имя в `UON_COMMENT_FIELD`.

Используется официальный метод добавления обращения:

```text
POST https://api.u-on.ru/KEY/lead/create.json
```

Обязательные данные отправляются как `application/x-www-form-urlencoded`: `source`, `u_name`, `u_phone`, `u_email` и настраиваемое поле комментария. Ограничение API U-ON Travel — не более 10 запросов в секунду.

## Настройка ВКонтакте

1. В настройках сообщества создайте ключ доступа с правом работы с сообщениями.
2. Включите сообщения сообщества.
3. В Callback API задайте адрес `https://ВАШ-ДОМЕН/vk/callback`.
4. Заполните в `.env` значения `VK_GROUP_ID`, `VK_CONFIRMATION_CODE`, `VK_CALLBACK_SECRET`, `VK_GROUP_TOKEN`.
5. Подтвердите сервер и включите событие **Входящее сообщение**.

## Email и Telegram

- Создайте API-ключ Resend, подтвердите домен отправителя и заполните `RESEND_API_KEY`, `NOTIFY_EMAIL_FROM`, `NOTIFY_EMAIL_TO`.
- Создайте Telegram-бота через BotFather, добавьте его в нужный чат и заполните `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`.
- Несколько email получателей перечисляются через запятую.

## Переменные окружения

| Переменная | Назначение |
|---|---|
| `VK_GROUP_ID` | ID сообщества VK |
| `VK_CONFIRMATION_CODE` | Строка подтверждения Callback API |
| `VK_CALLBACK_SECRET` | Секрет Callback API |
| `VK_GROUP_TOKEN` | Ключ доступа сообщества |
| `UON_API_KEY` | Ключ API U-ON Travel |
| `UON_SOURCE` | Название источника обращения |
| `UON_COMMENT_FIELD` | API-поле для результатов квалификации |
| `RESEND_API_KEY` | Ключ сервиса email-уведомлений |
| `NOTIFY_EMAIL_FROM` | Подтверждённый отправитель |
| `NOTIFY_EMAIL_TO` | Получатели через запятую |
| `TELEGRAM_BOT_TOKEN` | Токен Telegram-бота |
| `TELEGRAM_CHAT_ID` | ID чата для уведомлений |
| `DATA_FILE` | Файл состояния и дедупликации |

## Проверка

```bash
npm test
npm run check
```

Для приёмочного теста пройдите опрос в VK и убедитесь, что:

- карточка появилась в **«Обращениях»**, а не в «Заявках»;
- в карточке есть контакты и ответы туриста;
- создана только одна карточка;
- email и Telegram-уведомления содержат ID обращения;
- команда `заново` начинает новый независимый опрос.

## Безопасность

- Секреты хранятся только в `.env` и не коммитятся.
- Callback отклоняет запросы с неверным секретом или ID сообщества.
- Файл состояния создаётся с правами `0600`.
- В логи не выводятся API-ключи и полные ответы опроса.
