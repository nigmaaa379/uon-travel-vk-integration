# uon-travel-vk-integration

Интеграция чат-бота сообщества ВКонтакте с CRM U-ON Travel: создание обращений после квалификации туриста и уведомления куратора/руководителя.

## Раздел 1. Интеграция CRM-системы U-ON Travel с сообществом ВКонтакте

> Главное ограничение: не изменять существующую логику и последовательность сценариев чат-бота.

### 1. Подготовка и анализ

- [ ] Зафиксировать текущую логику сценария квалификации туриста.
- [ ] Определить точку успешного завершения опроса, после которой выполняется передача данных.
- [ ] Составить перечень данных туриста, собираемых чат-ботом.
- [ ] Получить и безопасно настроить учетные данные API U-ON Travel.
- [ ] Уточнить обязательные поля для создания карточки в разделе «Обращения».
- [ ] Согласовать получателей уведомлений: куратор и/или руководитель.
- [ ] Согласовать адреса электронной почты и целевой мессенджер.

### 2. API-интеграция с U-ON Travel

- [ ] Настроить API-клиент U-ON Travel.
- [ ] Настроить конфигурацию через переменные окружения без хранения токенов в коде.
- [ ] Реализовать проверку доступности и авторизации API.
- [ ] Подготовить маппинг данных чат-бота на поля обращения в CRM.
- [ ] Добавить валидацию обязательных данных перед отправкой.
- [ ] Реализовать обработку ошибок и безопасное логирование без персональных данных и секретов.

### 3. Создание обращения после квалификации

- [ ] Подключить отправку данных только после успешного завершения сценария квалификации.
- [ ] Реализовать создание новой карточки именно в разделе «Обращения».
- [ ] Проверить, что карточка не создается в статусе «Заявка».
- [ ] Сохранять в обращении все согласованные данные туриста и источник «ВКонтакте».
- [ ] Добавить защиту от повторного создания обращения при повторной обработке события.
- [ ] Реализовать повторные попытки при временной недоступности CRM.
- [ ] Убедиться, что сбой CRM не изменяет и не ломает существующий сценарий чат-бота.

### 4. Уведомления

- [ ] Настроить мгновенное email-уведомление при успешном создании обращения.
- [ ] Настроить мгновенное уведомление в согласованный мессенджер.
- [ ] Добавить в уведомление номер/ссылку на обращение и основные данные туриста.
- [ ] Не отправлять уведомление, если обращение фактически не создано.
- [ ] Реализовать обработку и журналирование ошибок доставки уведомлений.

### 5. Тестирование и приемка

- [ ] Проверить полный успешный путь: опрос → обращение → email → мессенджер.
- [ ] Проверить неполные и некорректные данные туриста.
- [ ] Проверить временную недоступность и ошибки API U-ON Travel.
- [ ] Проверить защиту от дублирования обращений.
- [ ] Провести регрессионное тестирование существующей логики чат-бота.
- [ ] Подтвердить создание карточки в «Обращениях», а не в статусе «Заявка».
- [ ] Зафиксировать результаты приемочного тестирования.
- [ ] Подготовить краткую инструкцию по настройке и эксплуатации интеграции.
